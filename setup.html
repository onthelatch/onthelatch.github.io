Setup Guide

For basic scanning functionality, you only need to log into the app with an account that has Scanning User permissions in Spektrix. However, for full functionality, we'll need to set up a few other things.

User Permissions

To use the attendee lists, scan reports, and group scanning features, your user account also needs Collect Report permissions. You can set this in the Spektrix Settings interface, under Users > User Accounts.

Report Configuration

The app's advanced features rely on a specific report. Go to the Insights and Mailings interface and set up a new Sales report named 'Ticket Scanning Report'.

The criteria set should be named 'Today's sales'. It needs to search for tickets with the Seat Status of 'Sold' or 'Scanned', and the event's Start Date (under 'Event Instances') should be for 'Today'.

The report output columns must be configured exactly as follows:

These output columns can be found in the following categories:

  • Events: Duration, Event Name
  • Event Instances: Event Date/Time, Event Instance ID, Seating Plan Web Name, Event Instance Capacity, Number Sold
  • Seats: Area, Scanned
  • Tickets/Reservations: Seat, Group Size, Ticket Attributes, Ticket Type, Barcode
  • Customers: Owner ID, Owner First Name, Owner Last Name

Report Schedule Setup

Next, we need to give your user accounts access to this report. In the Insights & Mailings interface, click 'Report Schedules' and then 'Add' to create a new schedule.

Name it something memorable like 'Ticket Scanning app report', check the 'Run on demand' box, and proceed to the next step to add the report.

Select the 'Ticket Scanning Report' you just created, using the 'Today's events' criteria. In the delivery details, ensure the name is 'Ticket Scanning Report' and add the email addresses of all scanning user accounts, separated by commas. Deliver as a Link with a file format of CSV.

Access Notes Feature

Finally, the 'Access Notes' feature requires a specific Ticket Attribute. In the Settings interface, under 'Attribute Definitions', click on 'Ticket' and add a new attribute called 'Accessibility Needs'.

For consistency, setting this up as a dropdown list is recommended. You can then define the specific notes your Front of House team will find most useful. Alternatively, you can use a text field, but a character limit is advised.

Once these steps are complete, you're ready to use all the features of the Ticket Scanner app!
